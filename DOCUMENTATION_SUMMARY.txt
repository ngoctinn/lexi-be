📊 CONVERSATION FLOW:

Phase 1: Load Scenarios
  └─ GET /scenarios

Phase 2: Create Session
  └─ POST /sessions

Phase 3: Main Loop (Repeat)
  ├─ User submits text
  ├─ Backend analyzes input
  ├─ Bedrock generates AI response (streaming)
  ├─ Polly synthesizes audio
  └─ POST /sessions/{id}/turns

Phase 4: Complete Session
  └─ POST /sessions/{id}/complete

Phase 5: Show Results
  └─ Display scoring

================================================================================

🔑 KEY ENDPOINTS:

GET  /scenarios                    - List all scenarios
POST /sessions                     - Create session
GET  /sessions                     - List user sessions
GET  /sessions/{id}                - Get session details
POST /sessions/{id}/turns          - Submit turn (main loop)
POST /sessions/{id}/complete       - Complete session

================================================================================

📊 DATA MODELS:

Session:
  - session_id, user_id, scenario_id
  - learner_role_id, ai_role_id, ai_gender, level
  - status (ACTIVE, COMPLETED, PAUSED)
  - total_turns, user_turns, hint_used_count
  - metrics: avg_ttft_ms, avg_latency_ms, total_cost_usd

Turn:
  - session_id, turn_index (0, 1, 2, ...)
  - speaker (USER or AI)
  - content, audio_url, translated_content
  - metrics (AI turns only): ttft_ms, latency_ms, output_tokens, cost_usd

Scenario:
  - scenario_id, scenario_title, context
  - roles (exactly 2), goals
  - difficulty_level (A1-C2)

================================================================================

⚠️ IMPORTANT NOTES:

1. Authentication: All endpoints require JWT token
2. Fallback: If Bedrock fails, returns "Thanks. Could you say a bit more?"
3. Metrics: Only on AI turns, not user turns
4. Audio URLs: S3 presigned URLs, valid 1 hour
5. Decimal Fields: Returned as strings, parse when needed
6. Timestamps: ISO 8601 format
